# LASTOP — исправленная структурированная версия проекта

Содержимое архива:
- `main.go` — точка входа
- `database/` — подключение и схема БД
- `handlers/` — обработчики API
- `middleware/` — авторизация
- `models/` — модели данных
- `utils/` — JWT-утилиты
- `web/` — статический фронтенд
- `docker-compose.yml` — запуск приложения и PostgreSQL
- `Dockerfile` — сборка контейнера
- `AUDIT.md` — подробный аудит исходных файлов и перечень исправлений

## Быстрый запуск

### Вариант 1. Через Docker
```bash
docker compose up --build
```

После запуска приложение будет доступно на:
- `http://localhost:8080`

### Вариант 2. Локально
1. Создайте `.env` на основе `.env.example`
2. Поднимите PostgreSQL
3. Запустите:
```bash
go mod tidy
go run .
```

Если используете внешний PostgreSQL (например Railway), достаточно указать `DATABASE_URL` в `.env`:
```bash
DATABASE_URL=postgresql://<user>:<password>@<host>:<port>/<db_name>
```
Приложение автоматически выставит `sslmode=require` для удалённого хоста, если параметр не указан в URL.


## Тестовый пользователь (создается автоматически)
При первом запуске PostgreSQL через `docker compose up --build` в БД добавляется тестовый профиль:

- Email: `test.user@lastop.local`
- Пароль: `TestPass123!`
- Имя: `Тестовый Пользователь`

Важно: `init.sql` выполняется только при первичной инициализации тома БД. Если том `postgres_data` уже создан, пересоздайте его, чтобы заново применить сиды:
```bash
docker compose down -v
docker compose up --build
```


## Деплой на Railway
1. Создайте новый проект на Railway и подключите этот репозиторий.
2. Railway автоматически использует `railway.json` и запустит сборку `go build -mod=mod -o main .`.
3. Добавьте переменные окружения из `.env.example` (минимум: `JWT_SECRET`, `CORS_ALLOWED_ORIGINS`).
4. Подключите PostgreSQL plugin в Railway (переменная `DATABASE_URL` добавляется автоматически).
5. После деплоя проверьте `https://<your-domain>/api/health` — должен вернуться `status: ok` или `status: degraded` при отключенной БД.

## Важное замечание
В текущей среде у меня не было доступа к внешнему Go module proxy, поэтому я не смог полностью прогнать финальную сборку через `go mod tidy` и `go build` именно внутри контейнера. Исходники проекта, структура и исправления подготовлены так, чтобы проект собирался в обычной среде с доступом к Go-зависимостям.

## Troubleshooting: go.sum
Если при запуске проверок появляется ошибка `malformed go.sum`, значит файл контрольных сумм был повреждён невалидным текстом.

В этом случае:
```bash
rm -f go.sum
go mod tidy
go test ./...
```

Команда `go mod tidy` пересоздаст корректный `go.sum` автоматически.
