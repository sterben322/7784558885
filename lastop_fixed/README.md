# LASTOP — исправленная структурированная версия проекта

Содержимое архива:
- `main.go` — точка входа
- `database/` — подключение и схема БД
- `handlers/` — обработчики API
- `middleware/` — авторизация
- `models/` — модели данных
- `utils/` — JWT-утилиты
- `web/` — статический фронтенд
- `docker-compose.yml` — запуск приложения и PostgreSQL
- `Dockerfile` — сборка контейнера
- `AUDIT.md` — подробный аудит исходных файлов и перечень исправлений

## Быстрый запуск

### Вариант 1. Через Docker
```bash
docker compose up --build
```

После запуска приложение будет доступно на:
- `http://localhost:8080`

### Вариант 2. Локально
1. Создайте `.env` на основе `.env.example`
2. Поднимите PostgreSQL
3. Запустите:
```bash
go mod tidy
go run .
```

## Важное замечание
В текущей среде у меня не было доступа к внешнему Go module proxy, поэтому я не смог полностью прогнать финальную сборку через `go mod tidy` и `go build` именно внутри контейнера. Исходники проекта, структура и исправления подготовлены так, чтобы проект собирался в обычной среде с доступом к Go-зависимостям.
